    • Lebar Bus
      Lebar bus menunjukkan jumlah bit data yang dapat ditransfer secara bersamaan dalam satu siklus. Semakin lebar bus, semakin banyak data yang dapat dikirim sekaligus sehingga kinerja sistem menjadi lebih cepat dan efisien.
    • Kecepatan Bus (Clock Speed)
      Kecepatan bus menentukan seberapa sering data dapat ditransfer dalam satu detik. Bus dengan frekuensi yang lebih tinggi mampu meningkatkan throughput data dan mempercepat komunikasi antar komponen komputer.
    • Jenis dan Arsitektur Bus
      Arsitektur bus memengaruhi cara perangkat berbagi jalur komunikasi. Bus lama yang digunakan bersama-sama oleh banyak perangkat cenderung mengalami kemacetan, sedangkan bus modern dengan arsitektur point-to-point lebih cepat dan efisien.
    • Jumlah Perangkat pada Bus
      Semakin banyak perangkat yang terhubung pada satu bus, semakin besar kemungkinan terjadinya perebutan akses. Hal ini menyebabkan perangkat harus menunggu giliran sehingga kinerja sistem dapat menurun.
    • Mekanisme Arbitrasi Bus
      Arbitrasi bus menentukan perangkat mana yang berhak menggunakan bus pada suatu waktu. Mekanisme yang baik dapat mengatur prioritas akses sehingga sistem tetap responsif dan efisien.
    • Overhead Protokol Bus
      Setiap transfer data memerlukan informasi tambahan seperti alamat dan sinyal kontrol. Overhead yang terlalu besar akan mengurangi bandwidth efektif yang tersedia untuk data utama sehingga kinerja bus menurun.
    • Teknologi Pendukung (DMA)
      Teknologi Direct Memory Access memungkinkan perangkat I/O mentransfer data langsung ke memori tanpa melibatkan CPU secara terus-menerus. Hal ini membuat penggunaan bus lebih efisien dan meningkatkan kinerja sistem secara keseluruhan.

  2. Direct Memory Access (DMA) adalah teknologi yang dirancang untuk mengatasi keterbatasan kinerja bus yang terjadi ketika CPU harus terlibat langsung dalam setiap proses perpindahan data antara perangkat input/output (I/O) dan memori utama. Pada sistem komputer tanpa DMA, setiap data yang dikirim dari perangkat I/O, seperti hard disk, kartu jaringan, atau sound card, harus terlebih dahulu melewati CPU. CPU membaca data dari perangkat, kemudian menuliskannya ke memori, atau sebaliknya. Proses ini menyebabkan CPU menjadi sangat sibuk, sementara bus sistem juga dipenuhi oleh lalu lintas data yang sebenarnya tidak berkaitan langsung dengan eksekusi instruksi program. Akibatnya, kinerja bus menurun dan CPU tidak dapat digunakan secara optimal untuk menjalankan tugas lain.

    Dengan adanya DMA, peran CPU dalam proses transfer data menjadi jauh lebih kecil. CPU hanya bertugas melakukan inisialisasi, yaitu menentukan alamat awal di memori, jumlah data yang akan ditransfer, serta arah transfer data. Setelah konfigurasi ini selesai, CPU menyerahkan kendali transfer sepenuhnya kepada DMA controller. DMA controller kemudian mengambil alih penggunaan bus dan melakukan transfer data secara langsung antara perangkat I/O dan memori utama tanpa harus melewati CPU. Selama proses ini berlangsung, CPU dapat tetap menjalankan instruksi-instruksi lain, sehingga tercipta kerja paralel antara CPU dan DMA. Hal inilah yang secara signifikan meningkatkan efisiensi penggunaan bus dan keseluruhan kinerja sistem.

    Peningkatan kinerja bus terjadi karena DMA mampu mengurangi jumlah akses CPU terhadap bus. Pada sistem tanpa DMA, CPU harus berulang kali mengakses bus untuk setiap unit data yang dipindahkan, sehingga bus menjadi cepat penuh dan menimbulkan bottleneck. Dengan DMA, transfer dilakukan dalam satu rangkaian operasi yang lebih terstruktur dan efisien, sering kali dalam bentuk blok data yang besar. Hal ini membuat penggunaan bus menjadi lebih optimal karena tidak terjadi lalu lintas data yang berulang-ulang dan tidak perlu. Selain itu, DMA juga mengurangi jumlah interrupt yang harus ditangani oleh CPU. Tanpa DMA, setiap kali perangkat I/O siap mengirim atau menerima data, CPU harus merespons interrupt tersebut. Dengan DMA, interrupt biasanya hanya dikirim satu kali setelah seluruh proses transfer selesai, sehingga beban CPU dan bus semakin berkurang.

    Selain efisiensi, DMA juga meningkatkan kecepatan transfer data. Karena DMA controller dirancang khusus untuk menangani operasi transfer, proses pemindahan data dapat dilakukan dengan kecepatan mendekati maksimum kemampuan bus dan perangkat keras yang terlibat. Hal ini sangat terasa pada aplikasi yang membutuhkan transfer data dalam jumlah besar dan berkelanjutan, seperti pembacaan file besar dari disk, streaming audio dan video, atau penerimaan paket data pada kartu jaringan. Tanpa DMA, bus akan sering terganggu oleh aktivitas CPU, sedangkan dengan DMA, aliran data dapat berjalan lebih stabil dan konsisten.

    Secara keseluruhan, teknologi DMA meningkatkan kinerja bus dengan cara meminimalkan keterlibatan CPU dalam transfer data, mengurangi kepadatan lalu lintas bus, memungkinkan pemrosesan paralel antara CPU dan perangkat I/O, serta mempercepat proses pemindahan data. Dengan mekanisme ini, bus tidak lagi menjadi hambatan utama dalam sistem komputer, dan sumber daya sistem dapat dimanfaatkan secara lebih efisien dan seimbang.

  3. Arsitektur Bus Jamak Tradisional

    Alur dimulai dari processor (CPU). Setiap kali CPU mengeksekusi instruksi dan membutuhkan data, ia pertama kali berkomunikasi melalui local bus. Local bus adalah jalur paling cepat karena langsung terhubung ke CPU. Pada tahap ini, CPU akan mengecek cache terlebih dahulu. Jika data yang dibutuhkan ada di cache, maka data langsung dikirim kembali ke CPU tanpa harus melibatkan bus lain, sehingga proses berlangsung sangat cepat.

    Jika data tidak ditemukan di cache, maka permintaan data diteruskan ke system bus. System bus berfungsi sebagai jalur utama yang menghubungkan CPU, cache, dan main memory (RAM). Melalui system bus inilah CPU mengambil data dari RAM atau menyimpan hasil proses ke memori utama. Cache juga terhubung ke system bus agar data yang sering digunakan bisa disalin dan disimpan lebih dekat ke CPU untuk akses berikutnya.

    Selain ke memori, CPU juga dapat berkomunikasi dengan perangkat input/output berkecepatan relatif tinggi melalui local I/O controller. Controller ini menjadi penghubung antara local bus dan system bus untuk menangani perangkat I/O tertentu tanpa membebani CPU secara langsung. Dengan cara ini, CPU tidak perlu mengatur detail teknis setiap perangkat.

    Untuk perangkat yang lebih lambat seperti modem dan serial, alurnya juga melalui expansion bus, namun biasanya lewat expansion bus interface terlebih dahulu. Interface ini berfungsi sebagai penyesuai kecepatan dan protokol, sehingga perangkat lambat tidak mengganggu lalu lintas data pada system bus maupun local bus.

    Arsitektur Bus Jamak Kinerja Tinggi

    Alur komunikasi dimulai dari processor (CPU). Ketika CPU membutuhkan data atau ingin mengirim data, pertama kali ia berkomunikasi melalui local bus. Local bus ini adalah jalur tercepat karena berada paling dekat dengan CPU. Jika data yang dibutuhkan masih tersedia di cache, maka data langsung diambil dari cache tanpa perlu keluar ke bus lain. Ini membuat proses sangat cepat dan mengurangi lalu lintas ke bagian sistem yang lebih bawah.

    Jika data tidak ada di cache, maka permintaan data diteruskan oleh cache/bridge ke system bus. System bus ini menjadi jalur penghubung antara CPU dan main memory (RAM). Di sinilah CPU mengambil atau menyimpan data ke memori utama. Setelah data dari RAM diperoleh, data dikirim kembali ke CPU melalui jalur yang sama, melewati bridge dan local bus.

    Pada saat yang sama, ketika terjadi proses yang melibatkan perangkat berkecepatan tinggi seperti VGA, video, LAN, atau SCSI, alurnya tidak langsung mengganggu CPUâ€“memori. Perangkat-perangkat tersebut berkomunikasi melalui high speed bus. High speed bus ini dihubungkan ke sistem lewat bridge, sehingga transfer data besar (misalnya tampilan grafis atau streaming data jaringan) dapat berjalan cepat tanpa membebani system bus secara berlebihan.

    Jika yang berkomunikasi adalah perangkat berkecepatan rendah seperti printer, modem, fax, atau port serial, maka alurnya diteruskan ke expansion bus melalui expansion bus interface. Data dari perangkat ini mengalir di expansion bus terlebih dahulu, lalu jika perlu diteruskan ke sistem utama, barulah melewati interface menuju bus di atasnya. Karena kecepatannya rendah, jalur ini dipisahkan agar tidak memperlambat bus cepat.
