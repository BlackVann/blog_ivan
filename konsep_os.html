Konsep Di Sistem Operasi

Batch System

Batch system adalah menjalankan pekerjaan perangkat lunak secara otomatis dan tanpa pengawasan. Pengguna menjadwalkan pekerjaan eperti pencadangan, pemfilteran, dan pengurutan dapat menjadi komputasi yang intensif dan tidak efisien untuk dijalankan pada transaksi data individual. Biasanya, pekerjaan dijadwalkan untuk dijalankan pada waktu yang telah ditentukan , saat suatu peristiwa terjadi seperti penghujung hari atau semalam, atau saat sumber daya komputer tersedia. Beberapa contoh sistem operasi pemrosesan batch meliputi:

  • z/OS milik IBM
  • Unisys MCP
  • Burroughs MCP/BCS

Sistem proses batch tersebut digunakan untuk memproses berbagai jenis data dan permintaan. Beberapa jenis pekerjaan pemrosesan batch yang paling umum meliputi:

  • Penagihan mingguan/bulanan
  • Penggajian
  • Pemrosesan inventaris
  • Pembuatan laporan
  • Konversi data
  • Siklus berlangganan
  • Pemenuhan rantai pasokan

Critical Section

Critical section adalah bagian penting dalam program yang tidak boleh dijalankan oleh dua orang (proses) sekaligus, karena bagian itu mengubah data yang sama.Biasanya, critical section mengakses sumber daya bersama, seperti struktur data , perangkat periferal, atau koneksi jaringan, yang tidak akan beroperasi dengan benar dalam konteks beberapa akses konkuren Berikut ini adalah beberapa contoh sistem dan library yang digunakan untuk mengatur critical section di berbagai bahasa pemrograman dan sistem operasi. Setiap sistem menyediakan mekanisme sinkronisasi berbeda untuk memastikan bahwa hanya satu proses atau thread yang dapat mengakses bagian critical pada satu waktu.

  • windows.h milik Windows API
  • pthread.h milik POSIX (Linux/UNIX)
  • java.util.concurrent milik Java
  • threading milik Python
  • System.Threading milik C# / .NET
  • sync milik Golang
  • std::sync milik Rust

Critical section memiliki beberapa bagian meliputi:

  • Bagian Entri (Entry Section)
    Proses meminta izin sebelum masuk ke critical section. Biasanya menggunakan alat sinkronisasi seperti mutex atau semaphore.
  • Critical section (Critical Section)
    (Critical Section) Bagian kode yang mengakses atau mengubah data bersama. Hanya satu proses boleh di sini dalam satu waktu.
  • Bagian Keluar (Exit Section)
    Setelah selesai, proses melepaskan izin (unlock / release semaphore) agar proses lain bisa masuk.
  • Bagian Sisa (Remainder Section)
    Bagian kode lain yang tidak berhubungan dengan sumber daya bersama, yang bisa dijalankan bebas.

Contoh critical section dalam Aplikasi Dunia Nyata

  • Sistem Perbankan (ATM atau Online Banking)
    • Critical section:Memperbarui saldo akun selama penyetoran atau penarikan.
    • Masalah jika tidak ditangani: Dua penarikan bersamaan dapat mengakibatkan saldo akhir yang salah karena kondisi balapan.
  • Sistem Pemesanan Tiket (Maskapai Penerbangan, Film, Kereta Api)
    • Critical section: Memesan kursi terakhir yang tersedia.
    • Masalah jika tidak ditangani: Dua pengguna mungkin diperlihatkan kursi yang sama dan keduanya memesannya, menyebabkan pemesanan berlebih.
  • Print Spooler di Printer Jaringan
    • Critical section: Mengirim pekerjaan cetak ke antrean printer.
    • Masalah jika tidak ditangani: Pekerjaan cetak dapat tercampur atau terlewati jika beberapa pengguna mengirim pekerjaan secara bersamaan.
  • Pengeditan File dalam Dokumen Bersama (misalnya, Google Docs, MS Word dengan akses bersama)
    • Critical section: Menyimpan atau menulis ke dokumen bersama.
    • Masalah jika tidak ditangani: Pengeditan yang dilakukan secara bersamaan dapat menyebabkan versi yang saling bertentangan atau hilangnya data.

Process Control Block

Process Control Block (PCB), atau disebut juga deskriptor proses, adalah struktur data yang digunakan oleh sistem operasi untuk menyimpan semua informasi mengenai suatu proses. Ketika sebuah proses dibuat atau dijalankan, sistem operasi akan membentuk PCB yang berisi data seperti identitas proses, status proses (baru, siap, berjalan, menunggu, atau dihentikan), lokasi memori, serta informasi CPU terakhir. PCB ini berperan penting dalam pelacakan proses dan peralihan konteks, karena memungkinkan sistem untuk menghentikan sementara satu proses dan melanjutkan proses lainnya tanpa kehilangan data. Kernel sistem operasi menyimpan seluruh PCB dalam tabel proses, yang mencatat semua proses aktif di dalam sistem.

Hampir semua sistem operasi modern saat ini telah memiliki Process Control Block, meskipun struktur dan detail isinya berbeda-beda pada setiap sistem operasi. Perbedaan inilah yang membuat PCB memiliki sifat unik di setiap sistem, namun tetap memiliki tujuan utama yang sama — yaitu mengelola dan mengawasi jalannya proses secara efisien di dalam komputer.

Distributed Processing

Distributed processing adalah proses mengambil tugas komputasi yang kompleks dan membaginya ke dalam beberapa bagian yang lebih kecil, lalu mendistribusikannya ke sejumlah mesin (node) dalam jaringan. Setiap node memproses bagiannya masing-masing, kemudian hasilnya dikirim kembali ke sistem pusat untuk dikompilasi menjadi satu keluaran yang utuh dan mulus.

Pendekatan ini memungkinkan sistem untuk menyelesaikan tugas dengan lebih cepat dan efisien dibandingkan jika semua proses dijalankan hanya pada satu komputer tunggal. Selain itu, distributed processing juga meningkatkan keandalan (reliability) dan skalabilitas (scalability), karena jika satu node mengalami kegagalan, node lain masih dapat melanjutkan pekerjaan tanpa menghentikan seluruh sistem. Setiap node bertanggung jawab untuk memproses bagian tugasnya sendiri secara paralel dan mandiri, menggunakan sumber daya komputasinya masing-masing. Setelah semua node menyelesaikan bagian yang menjadi tanggung jawabnya, hasil dari setiap node dikirim kembali ke sistem pusat atau pengendali utama untuk digabungkan (dikompilasi) menjadi satu keluaran yang utuh, konsisten, dan mulus.

Thread

Thread adalah unit-unit kecil program komputer yang dapat berjalan secara independen. Thread memungkinkan program untuk menjalankan beberapa tugas secara bersamaan, misalnya menjalankan beberapa bagian program secara bersamaan. Hal ini membuat program lebih efisien dan responsif, terutama untuk tugas-tugas yang dapat dibagi menjadi bagian-bagian yang lebih kecil. Ada dua jenis utama thread, yaitu User Level Thread dan Kernel Level Thread

  • User Level Thread : Thread yang dikelola sepenuhnya oleh program (di level pengguna), bukan oleh sistem. Contohnya thread pada bahasa Java atau POSIX Thread (pthreads).
  • Kernel Level Thread : Thread yang dikelola langsung oleh sistem operasi (kernel). Contohnya thread di sistem Windows, Linux, atau macOS modern.

Secara umum thread memiliki 2 cara kerja yaitu :

  • Single Thread : semua tugas dijalankan secara berurutan, satu per satu. Cocok untuk program sederhana seperti kalkulator atau program konsol.
  • Multi Thread : tugas bisa berjalan secara bersamaan (paralel). Cocok untuk aplikasi besar seperti browser, game, atau server web.
